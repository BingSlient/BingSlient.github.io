Kali linux 使用国内源

在国内使用 Kali Linux 进行更新的时候总是出现很多小问题，速度也慢，最简单的方法就是使用国内提供的软件源，虽然官方不推荐这么干，但是实际只要软件源可靠也没什么问题。

步骤

  1. 修改 /ect/apt/sources.list 文件如下：

注：

deb xxx kali-rolling main non-free contrib 为 Kali Linux 的常规软件源

deb-src xxx kali-rolling main non-free contrib 为提供软件源码的源

其中 kali-rolling 为 Kali Linux 的代号，这个代号要和你使用的 Kali Linux 系统对应，使用命令lsb_release -a 可以查看 Kali Linux 的代号。

  1. 更新软件源和软件

注：

apt-get update 用于同步远程仓库中的软件列表，以便让 apt-get 知道软件的最新状态，是否可更新。

apt-get upgrade 用于更新所有已经在系统安装的软件，如果更新软件需要更改其它软件的状态，如需要降级，则不更新软件。

apt-get dist-upgrade 和 apt-get upgrade 相似，但是会解决更新软件可能带来的依赖问题，因此有可能会删除一些包，安装额外的包等。

apt-get -s 选项可以模拟后续命令的执行，不会发生实际操作，所以可以先带 -s执行更新操作，查看哪些软件需要更新，更新会安装哪些包等。
